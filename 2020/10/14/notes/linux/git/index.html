    git工作模型

    新建

    在当前目录新建一个Git代码库

    $ git init

    新建一个目录，将其初始化为Git代码库

    $ git init [project-name]

    下载一个项目和它的整个代码历史

    $ git clone [url]

    配置

    生成key,用于git配置免密push

    $ ssh-keygen -t rsa -C "xxx@xx.com"

    显示当前配置列表

    $ git config --list

    设置提交代码时的用户信息

    $ git config --global user.name "xxx"
$ git config --global user.email "xxx@xx.com"

    设置当前仓库用户信息

    $ git config --local user.name "xxx"
$ git config --local  user.email "xxx@xx.com"

    工作区与暂存区交互(索引)

    添加到暂存区

    $ git add [文件或目录] ...

    添加前需要确认

    $ git add -p

    删除工作区文件, 且将删除放入暂存区

    $ git rm [文件或目录] ...

    停止追踪指定文件，但该文件会保留在工作区

    $ git rm --cached [文件]

    移动文件或目录(改名)

    $ git mv [旧名] [新名]

    暂存区与本地仓库交互

    提交到仓库

    $ git commit [文件或目录] -m [注释评论]

    提交时显示diff信息

    $ git commit -v

    使用一次新的commit, 替代上一次提交
    如果代码没有变化, 则用来改写上一次commit的提交信息 

    $ git commit --amend -m [注释评论]

    分支

    branch

    语法

    $ git branch [选项] [参数]

    选项

    • -r
      列出所有远端分支
    • -a
      列出所有本地分支和远端分支
    • -d [本地分支]
      删除分支
    • –track [本地分支] [远端分支]
      新建一个分支, 与指定的远程分支建立追踪关系
    • –set-upstream [本地分支] [远端分支]
      建立追踪关系, 在现有分支和指定的远端分支之间
    • -vv
      查看本地分支和远端分支的关联信息

    参数

    • 分支名
      要列出或新建的分支名

    示例

    删除远端分支 

    $ git push origin --delete [远端分支]

    或

    $ git branch -dr [remote/branch]

    checkout

    切换到指定分支, 并更新工作区

    $ git checkout [分支名]

    切换到上一个分支

    $ git checkout -

    merge

    合并指定分支到当前分支

    $ git merge [本地分支]

    标签

    新建一个tag在当前commit

    $ git tag [tag]

    新建一个tag在指定commit

    $ git tag [tag] [commit]

    删除本地tag

    $ git tag -d [tag]

    查看tag信息

    $ git show [tag]

    删除远端tag 

    $ git push origin :refs/tags/[tag名]

    提交指定tag

    $ git push [remote] [tag]

    提交所有tag

    $ git push [remote] --tags

    远端

    本地仓库关联/取消关联远端仓库

    git remote add origin git@github.com:changdingfang/test.git
git remote remove origin

    下载远端仓库的所有变动 

    $ git fetch [remote]

    显示远端仓库

    $ git remote -v

    显示远端仓库的信息

    $ git remote show [remote]

    增加一个新的仓库并命名

    $ git remote add [名字] [url]

    取回远程仓库的变化，并与本地分支合并

    $ git pull [remote] [branch]

    拉取远程仓库时，忽略不相关历史的

    $ git pull [remote] [branch] --allow-unrelated-histories

    上传本地指定分支到远程仓库

    $ git push [remote] [branch]

    强行推送当前分支到远程仓库, 即使有冲突(尽量不要这样做)

    $ git push [remote] --force

    推送所有分支到远程仓库

    $ git push [remote] --all

    信息

    status

    显示有变更的文件

    $ git status

    log

    语法

    $ git log [选项]

    选项

    • 显示过去n次的提交提示
    • -S [keyword]
      根据关键字搜索提交历史
    • –stat
      显示每次提交的文件修改统计信息。
    • –shortstat
      只显示 –stat 中最后的行数修改添加移除统计。
    • –name-only
      仅在提交信息后显示已修改的文件清单。
    • –name-status
      显示新增、修改、删除的文件清单。
    • –abbrev-commit
      仅显示 SHA-1 校验和所有 40 个字符中的前几个字符。
    • –relative-date
      使用较短的相对时间而不是完整格式显示日期（比如“2 weeks ago”）。
    • –graph
      在日志旁以 ASCII 图形显示分支与合并历史。
    • –pretty
      使用其他格式显示历史提交信息。可用的选项包括 oneline、short、full、fuller 和 format（用来定义自己的格式）。
    • –oneline
      –pretty=oneline –abbrev-commit 合用的简写。

    使用示例

    显示某个commit之后的所有变动, 每个commit占据一行

    $ git log [tag] HEAD --pretty=format:%s

    显示某个commit之后的所有变动，其”提交说明”必须符合搜索条件

    $ git log [tag] HEAD --grep feature

    显示某个文件的版本历史，包括文件改名

    $ git log --follow [file]
$ git whatchanged [file]

    显示指定文件相关的每一次diff

    $ git log -p [file]

    显示过去5次提交, 且仅显示简略信息

    $ git log -5 --pretty --oneline

    显示所有提交过的用户，按提交次数排序

    $ git shortlog -sn

    blame

    显示指定文件是什么人在什么时间修改过

    $ git blame [file]

    diff

    显示暂存区和工作区的差异

    $ git diff

    显示暂存区和上一个commit的差异

    $ git diff --cached [file]

    显示工作区与当前分支最新commit之间的差异

    $ git diff HEAD

    显示两次提交之间的差异

    $ git diff [first-branch]...[second-branch]

    显示今天你写了多少行代码

    $ git diff --shortstat "@{0 day ago}"

    show

    显示某次提交的元数据和内容变化

    $ git show [commit]

    显示某次提交发生变化的文件

    $ git show --name-only [commit]

    显示某次提交时，某个文件的内容

    $ git show [commit]:[filename]

    显示当前分支的最近几次提交

    $ git reflog

    临时存储代码

    增加

    存储代码，压入代码堆栈

    $ git stash

    或(带有注释消息)

    $ git stash save "message"

    查看

    查看现在有点记录

    $ git stash list

    检查不一样的地方

    $ git stash show

    删除

    弹出栈顶的代码

    $ git stash pop

    删除指定

    $ git stash drop stash@{1}

    删除所有

    $ git stash clear

    撤销

    恢复本地仓库的指定文件或目录到工作区(默认最新的一次提交)

    $ git checkout [文件或目录] ...

    恢复本地仓库某个commit的指定文件到暂存区和工作区

    $ git checkout [commit] [file]

    重置暂存区的指定文件, 与上一次commit保持一致，但工作区不变

    $ git reset [file]

    重置暂存区与工作区, 与上一次commit保持一致

    $ git reset --hard

    重置当前分支的指针为指定commit，同时重置暂存区，但工作区不变

    $ git reset [commit]

    重置当前分支的HEAD为指定commit，同时重置暂存区和工作区，与指定commit一致

    $ git reset --hard [commit]

    重置当前HEAD为指定commit，但保持暂存区和工作区不变

    $ git reset --keep [commit]

    新建一个commit，用来撤销指定commit
    后者的所有变化都将被前者抵消，并且应用到当前分支 

    $ git revert [commit]

    暂时将未提交的变化移除，稍后再移入

    $ git stash
$ git stash pop
